How it works

An image library that grows as more people use it

Your request flows down, your image comes straight back up — and every new image joins the shared library, so the next person's prompt is more likely to be a hit.

Your request
"a vintage bicycle against a brick wall"
POST /v1/images/generations
The response
https://cdn.wagmiphotos.dev/img/pd12m-8f31…
"result": "hit" · 41 ms · $0.04 saved

Cloudflare edge worker — authenticates your key, CLIP-embeds the prompt, and answers from the edge. It never runs a GPU in the request path.

The shared library
  1. 1Your prompt's CLIP vector is matched against every image in the library.
  2. 2Near match — the image is served instantly. That's a hit.
  3. 3Truly new — you get a 202 and the prompt joins the build queue. No waiting.
  4. 4The backfill generates the most-requested misses, so tomorrow they're hits.

The API

One endpoint. Cache included.

Point the official OpenAI SDK at a new base URL — no other changes. You get back the OpenAI response shape plus a shared_cache block telling you whether it was a hit, how close the match was, and what it saved you.

Try it in the playground →
POST /v1/images/generations 200 OK · 41 ms
Request 
curl -X POST https://wagmiphotos.dev/v1/images/generations \
  -H "Authorization: Bearer $WAGMI_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a vintage bicycle against
               a brick wall",
    "model": "shared-cache-gmicloud-
              flux-schnell",
    "cache_tolerance": 0.15
  }'
from openai import OpenAI

client = OpenAI(
    base_url="https://wagmiphotos.dev/v1",
    api_key="sc-your-key",
)

img = client.images.generate(
    model="shared-cache-gmicloud"
          "-flux-schnell",
    prompt="a vintage bicycle "
           "against a brick wall",
    extra_body={
        "cache_tolerance": 0.15},
)
print(img.data[0].url)
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://wagmiphotos.dev/v1",
  apiKey: "sc-your-key",
});

const img = await client.images
  .generate({
    model: "shared-cache-gmicloud" +
           "-flux-schnell",
    prompt: "a vintage bicycle " +
            "against a brick wall",
    cache_tolerance: 0.15,
  });
console.log(img.data[0].url);

Legal & Usage Policy

Effective Date: July 1, 2026

1. Privacy Policy

WagmiPhotos is designed from the ground up to respect developer and user privacy:

  • Local-Only Key Storage: All API keys you input (OpenAI, Gemini, GMICloud, and Hugging Face tokens) are saved strictly inside your browser's local localStorage. They never touch our database servers in raw form.
  • Secure Telemetry: To track metrics without exposing developers, keys used in requests are instantly hashed (SHA-256) on the backend. No raw keys or profile emails are stored in our analytics database.
  • Encrypted Transit: All communication between the browser client, the proxy server, and the AI providers is fully encrypted via HTTPS.

2. Terms of Use & Data Licensing

By using WagmiPhotos, you agree to comply with the terms of our backend caching logic and the licenses of our creative catalogs:

  • Public Domain Data: WagmiPhotos leverages the PD12M dataset from Spawning. This dataset is licensed under the CDLA-Permissive-2.0 agreement. You are permitted to copy, distribute, modify, and utilize all pre-seeded images for commercial or personal purposes without attribution.
  • Proxy Fair Use: You agree not to abuse the cache endpoints through coordinated denial-of-service, scraping, or cache pollution. Misses are generated on demand and subject to your own provider token limits.
