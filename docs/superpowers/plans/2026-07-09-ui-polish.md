# UI Polish Batch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Six SPA polish items: styled ToS "Log out" button, star badge hidden while the repo is private, site-consistent form controls on the BYOK/collections cards, playground reordered with tolerance gated behind the generate checkbox, friendlier wording, and a key-status indicator.

**Architecture:** All changes live in `projects/worker/public/index.html` (single-file SPA). One new CSS class (`.field-input`), one extended CSS selector, markup edits at known anchors, and two small JS additions (`updateGenGating`, badge hide/show). No worker/API changes.

**Tech Stack:** Single-file SPA (inline CSS/JS), vitest for the router smoke tests, wrangler dev for visual verification.

**Spec:** `docs/superpowers/specs/2026-07-09-ui-polish-design.md` (approved 2026-07-09).

## Global Constraints

- ONLY `projects/worker/public/index.html` changes (Tasks 1–3). No worker source, no contract, no migrations.
- Wording, exact: section label **"When there's no close match"**; checkbox lead **"Create a fresh image."** with sub-copy "If the library has nothing close enough, a new image is made for your prompt and added to the library."; tolerance label **"Match strictness"**; slider ends **"Exact match"** / **"Loose match"**. The numeric "Tolerance: N" info line stays.
- Tolerance gating: checkbox unchecked → tolerance group at opacity 0.45 with the slider `disabled`, and the POST body OMITS `cache_tolerance` entirely (server default 0.15). Checked → behaves exactly as today.
- Key indicator states (from `currentByok`): enabled → `✓ Uses your <Provider> key ••••<last4> — <used>/<cap> images this month · Manage`; disabled → `⚠ Your <Provider> key is disabled — fix it in Account`; null → `No provider key set — your prompt joins the shared generation queue instead (can take a while). Add your key →`. All link to `#/account`. Hidden while the checkbox is unchecked. `key_last4` is escaped via `escapeHtml`.
- Verification per task: `npx vitest run test/router.test.ts` (32/32), script-parse check (`node -e "const m=require('fs').readFileSync('public/index.html','utf8').match(/<script>([\s\S]*)<\/script>/); new Function(m[1]); console.log('script parses')"`), full `npx vitest run` (319) + `npx tsc --noEmit`.
- Working directory: `projects/worker`. Commit after each task.

- [ ] **Step 2: Add `updateGenGating()`**

Next to `loadPlaygroundCollections()` in the script, add:

```js
    // Tolerance only matters when a miss can create something: greyed +
    // default when "create a fresh image" is off. Also shows which key a
    // fresh generation would use (or that the shared queue applies).
    function updateGenGating() {
      const on = document.getElementById('gen-on-miss')?.checked ?? true;
      const tolGroup = document.getElementById('tol-group');
      const tol = document.getElementById('tol');
      if (tolGroup) tolGroup.style.opacity = on ? '' : '0.45';
      if (tol) tol.disabled = !on;
      const el = document.getElementById('gen-key-status');
      if (!el) return;
      if (!on) { el.innerHTML = ''; return; }
      if (currentByok && currentByok.enabled) {
        const name = currentByok.provider === 'openai' ? 'OpenAI' : 'GMI Cloud';
        el.innerHTML = `<span style="color:var(--success);">✓</span> Uses your ${name} key <span style="font-family:var(--font-mono);">••••${escapeHtml(currentByok.key_last4)}</span> — ${currentByok.used_this_month}/${currentByok.monthly_cap} images this month · <a href="#/account" style="color:var(--red);">Manage</a>`;
      } else if (currentByok) {
        const name = currentByok.provider === 'openai' ? 'OpenAI' : 'GMI Cloud';
        el.innerHTML = `⚠ Your ${name} key is disabled — <a href="#/account" style="color:var(--red);">fix it in Account</a>.`;
      } else {
        el.innerHTML = `No provider key set — your prompt joins the shared generation queue instead (can take a while). <a href="#/account" style="color:var(--red);">Add your key →</a>`;
      }
    }
```
